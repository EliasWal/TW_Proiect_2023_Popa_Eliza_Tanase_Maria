1. Introduction

1.1 Purpose

The purpose of this report is to present a detailed description of the "Baby manager" web application. It will explain the purpose and features of the application, the interfaces and what the application will do.

1.2 Product scope

This application will be a Web Baby Info Manager designated to help parents keep track of their children's most important information. It is designed to make the task of being a parent easier by providing tools to assist in scheduling the every day life, as well as saving resources and documenting about their children's needs. By providing these tools, the application will meet parents' needs while remaining easy to understand and use.

More specifically, the "Baby Manager" application is constructed to allow a parent to manage and save information and multimedia resources about timetables, medical histories, relationships with other children and personal journal entries.

The goal of this application is to offer its users a safe place to stack their information concerning raising a child and to access them at any time, from any device.

1.3 References

The application uses a library to style a "Hamburger Menu", which can be identified by three horizontal lines, that appears in the topbar section when the screen size is smaller (than 915 px). By clicking on it, the menu for navigating the application will appear on the right side of the screen, and by clicking the "X" icon, it will disappear. The reason to use this feature comes from the need to create a responsive design and to have a pleasant interface for any of the application's users, regarding of the device they use. The library can be found at https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.0.0-beta2/css/all.min.css .

2. Overall description

2.1 Product functions

Functions you can access without an account:

Home function brings a user back to the initial page, which presents the scope and tools of the application.

Sign up function allows a guest to create an account so that they can use the features offered and their information can be stored in the data base.

Log in functions grants access to a user that already has an account to their personal page and information.

Functions you can access with an account:

Home function brings a user back to the initial page, which introduces the scope and tools of the application and presents a tutorial on how to navigate the application.

Log out function results in going back to the guest Welcome page, with the user no longer logged in their account.

Settings function allows the user to make any changes to their personal profile.

  • General settings
  • Manage your children
  • Change password

Memories function displays in a pleasing way the journal-entries-like blocks, each containing a picture and a relevant description.

  • Choose a child from the leftbar menu
  • Add a memory
  • Edit a memory
  • Delete a memory
  • Share a memory on the Social Media platforms

Calendars function offers the user a way to organize their children's feeding and sleeping timetables.

  • Choose a child from the leftbar menu
  • Add a new entry in the calendar
  • Edit the time or a note
  • Mark an activity as checked/unchecked

Friends function presents to the user any relationship that their children might have.

  • Add a new friend
  • Edit a relationship
  • Delete a relationship

Medical function saves the medical history of the user's children.

  • Choose a child from the leftbar menu
  • Add a new record
  • Edit a record
  • Delete a record

Media function offers the user a way to see and upload files for having them in one place.

  • Upload files like photos, videos, audio or any type of document
  • Edit an existing file
  • Delete unwanted file

2.2. User classes and Characteristics

The Web application "Baby Manager" is intended for the parents and parents-to-be who are looking for management and organizing features for their children's journeys.

A guest user is a person that has their first encounter with the application or a person that is not logged in. They have access to the "Welcome" page, where they can learn information about what they could benefit once they create an account. The page is designed so that the guest user will be well documented about the features and their uses, as to make a right decision whether or not to join. A guest user also has the possibility to sign up and become a logged user. By trying to log in without an account already created, the user will get a warning about their credentials not being in the database. A guest user doesn't have access to any of the actual features for managing their children's information. Parents or soon to be parents that are searching for applications about raising children may fit in this category.

A logged user is a person that has previously created an account by signing up and used their credentials to log in their said account. They have access to all the pages involved in the process of navigating the application, including: Home, Settings, Memories, Calendars, Friends, Media and Medical. A logged user is the only one that can save and manage their information within the "Baby manager" application. Every time they enter their account again, the information will be loaded from the database. They can also, at any point, log out of their account and become a guest user, with no way of accessing data until they log in back again. For the most part, the application is dedicated to this class of users, that will frequently visit the pages and interact with them. Parents and some soon to be parents that are searcing and/or currently using applications about raising children may fit in this category.

2.3 Operating environment

The application "Baby Manager"'s goal is to be implemented as to be used correctly and safely by any type of users, from any devices.

The main domanin will be hosted on a server for deliviring the website to users. The application will be operated on a Web browser. The adaptive features that were implemented are available on any of the well-known browsers. "Baby manager" will run on any operating system, since the application is independent of it, with minimum specification required.

PHP will be used to make the Web pages dynamic and interactive, as well as to work with a database. The application will use a PHP script to connect to and edit the SQL database, so that information can be stored and loaded.

2.4 Dependencies

The application is build with the scope of managering a great amount of information that the user inputs. And so it is dependent on a environment which allows working with a database, such as MySQL.

3. User interfaces

The "Baby Manager" web application intends to create a pleasing and easy to navigate user interface, as well as clear and concise.

The initial "Welcome" page, as well as all the other pages, have an aesthetically pleasing appearence, with bright colors like purple and pink, to create an almost childish look. Here the user can find the buttons "Sign up" and "Log in" which are vital for connecting to an account and using the features. "Sign up" is used to create a new account, whereas "Log in" is used to enter an already created account.

Another important tool that the user can find in every page is the topbar. Located at the start area of the page, it contains features with links for navigating the whole application. The user can access Memories, Calendars, Friends, Medical, Media, Home and Settings, as well as press the now "Log out" button to disconnect from their account.

By pressing the wheel-like image next to the "Log out" button, the user accesses their personal profile settings. They can, then, choose from the leftside menu, whether they want to manage their general settings, their children's information or change their password.

Each of the main pages "Memories", "Calendars", "Media", "Medical" have an introductive page to describe the tools that will be used and what is the scope of them. The user will, in consequence, have at their disposal a short documentation prepared for navigating easily.

In the left menu, the user can choose, from their list of children, which one's page and information they want to see. After choosing the prefered child in any of the pages "Memories", "Calendars" and "Medical" another page of information will be accessed.

For the "Friends" page, as a difference, all the data of the user's children is stored in one page, as well as for the "Media" page. In the "Friends" page, in the leftside menu, they can open the list of friends, as well as to add a new friend.

All of the beforesaid pages have some of the buttons used for adding, editing, saving, uploading, deleting or sharing on Social Media platforms the prefered information, through forms.

In what concerns responsiveness, the pages were carefully created and updated for any possible device. For example, for the "Medical" page, if we are to talk about a smaller size, the line with the record data becomes smaller and so a "More" page, which shows the information unfiltered, is needed.

4. System features

4.1 Home

4.1.1 Description

The "Home" page, whether it is seen by a guest user or a logged user, provides a certain level of details about the other features offered by the application.

4.1.2 Response Sequence

It can be accessed by clicking the "Home" hyperlink.

4.1.3 Functional Requirements

The page will be loaded on the browser every time the application is accessed or a user logges in.

4.2 Sign up

4.2.1 Description

The "Sign up" feature allows a user that doesn't have an account to create one and to benefit of all the managing features.

4.2.2 Response Sequence

It can be accessed by clicking the "Sign up" button. This operation will result in loading a page with a form. For creating an account, it is mandatory that the user fills in all the personal data in the form's white spaces or chooses from the examples added in the dropdown list. By clicking on the "Create an account" button, the registration will be finished and the user will be automatically logged.

4.2.3 Functional Requirements

The user's personal information will be safely saved in a database and used later to verify the user's log in. If the same email address or username have been used for another account registered on the application, an error will occur in the signing up process. The user will be notified about this problem.

4.3 Log in

4.3.1 Description

The "Log in" feature allows a user with an already created account to access it.

4.3.2 Response Sequence

It can be accessed by clicking the "Log in" button. This operation will result in loading a page with a form. For logging in an account, it is mandatory that the user completes their credentials (username and password) in the form's white spaces. By pressing the "Log in" button in the form page, the user will be redirected on the members' "Welcome" page.

4.3.3 Functional Requirements

The user's credentials will be verified in the process of searching them in the application's database. If they coincide with one of the user's that have previously signed up, then they will have access to the logged page. Their information will be loaded from the database. If there is no matching, an error will occur in the process of logging in. The user will be notified about this problem.

* A logged in user can exit their account by pressing the "Log out" button.

4.4 Settings

4.4.1 Description

The "Settings" feature's purpose is allowing the user to manage their personal profile information.

4.4.2 Response Sequence

It can be accessed by a logged user by clicking the wheel-like button [].

The "General" settings consist of a form that already has the user's data loaded, but they can't be modified, having the "read-only" activated. In this page they can visualize their general informations and press the "Edit" button to modify any of their personal information. After they click on "Edit", they will be redirected to another form, where the fields are editable, and save the info's update by pressing on save. All the new info will be replaced in the database.

The "Manage your children" settings is constructed by multiple sections for each of the user's children. Once again, they can "Edit" the information for a child, and even "Delete" them. They can also "Add" a new child by inserting their data in a similar form.

The "Change your password" feature is created to do so. The new password that the user chooses will be the only one used after that to log in their account.

The changes that the user makes to their personal data are irreversible and any other previous inputs will be overwrited.

4.4.3 Functional Requirements

Any change in the user's profile will be added to the database and the old, not valid information will be deleted. If the user tries to enter their account using old credentials, the database will consider their login information as a new user that doesn't have an account.

4.5 Memories

4.5.1 Description

In the page "Memories" the user can see the journal entries that are considered to be "memories".

4.5.2 Response Sequence

The "Children" list in the left bar allows the user to choose a child's page.

The user can "Add" a new journal entry by introducing some prefered image and description, as well as a date for the memory.

When the page is loaded, the user can see only a part of the description, for a good management of the space. When the user hovers over the journal entry, the whole description becomes visible.

For each of the "memories", the user can choose to "Edit", "Delete" or "Share" the information.

4.5.3 Functional Requirements

The journal entries will be saved in the database and loaded. Every change will be immediately seen by the user in the main page.

4.6 Calendars

4.6.1 Description

The "Calendars" page is an interactive timetable for the useful activities of eating and sleeping.

4.6.2 Response Sequence

The "Children" list in the left bar allows the user to choose a child's page.

The user can "Add" a new calendar input for their prefered activities, which consist in another form that let you set a time and some notes.

For every line of the calendar, the time and notes can be managed only after pressing the "Edit" button. Also, one of the two activities can be marked as done/not done with the use of checkboxes any time.

4.6.3 Functional Requirements

The calendar information will be saved in the database and loaded. Every change will be immediately seen by the user in the main page.

4.7 Medical

4.7.1 Description

The "Medical" page saves records of any health problems of the child in a "Medical History" template.

4.7.2 Response Sequence

The "Children" list in the left bar allows the user to choose a child's page.

The user can "Add" input for a new medical record, including a document of any kind.

For every line of the table, the information can be managed by clicking "Edit".

If the screen has a lower resolution, clicking on the 3 dots will alow the users to see all the informations or to edit them.

4.7.3 Functional Requirements

The medical records will be saved in the database and loaded. Every change will be immediately seen by the user in the main page.

4.8 Friends

4.8.1 Description

The "Friends" page shows a list of any relationships the user's children might have.

4.8.2 Response Sequence

The user can "Add" a new friend, by stating their name, relationship with the child, as well as including a reference photo.

For every friend in the list, the information can be "Edit"-ed or "Delete"-ed.

4.8.3 Functional Requirements

The friends list will be saved in the database and loaded. Every change will be immediately seen by the user in the main page.

4.9 Media

4.9.1 Description

The "Media" page shows already uploaded files like photos, videos and audio files.

4.9.2 Response Sequence

The user can "Upload" a new file, by stating the wanted title and uploading from his PC the file.

For each uploaded file, the information can be "Edit"-ed or "Delete"-ed using a form.

4.9.3 Functional Requirements

The uploaded files will be saved in the database and loaded from it. Every change will be immediately seen by the user in the main page.

5. Security requirements

The information received from the user will be encrypted and only after that saved in the database.

The personal data loaded from the database will be available only after checking the authenticity of the user by comparing the credentials offered in the log in stage with the credentials of existing users.

The main part of the application will not be available to guest users, who don't own or are not logged in an account.
