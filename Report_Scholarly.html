1. Introduction

1.1 Purpose

The purpose of this report is to present a detailed description of the "Baby manager" web application. It will explain the purpose and features of the application, the interfaces and what the application will do.

1.2 Product scope

This application will be a Web Baby Info Manager designated to help parents keep track of their children's most important information. It is designed to make the task of being a parent easier by providing tools to assist in scheduling the every day life, as well as saving resources and documenting about their children's needs. By providing these tools, the application will meet parents' needs while remaining easy to understand and use.

More specifically, the "Baby Manager" application is constructed to allow a parent to manage and save information and multimedia resources about timetables, medical histories, relationships with other children and personal journal entries.

The goal of this application is to offer its users a safe place to stack their information concerning raising a child and to access them at any time, from any device.

1.3 References

The application uses a library to style a "Hamburger Menu", which can be identified by three horizontal lines, that appears in the topbar section when the screen size is smaller (than 915 px). By clicking on it, the menu for navigating the application will appear on the right side of the screen, and by clicking the "X" icon, it will disappear. The reason to use this feature comes from the need to create a responsive design and to have a pleasant interface for any of the application's users, regarding of the device they use. The library can be found at https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.0.0-beta2/css/all.min.css .

2. Overall description

2.1 Product functions

Functions you can access without an account:

Home function brings a user back to the initial page, which presents the scope and tools of the application.

Sign up function allows a guest to create an account so that they can use the features offered and their information can be stored in the data base.

Log in functions grants access to a user that already has an account to their personal page and information.

Functions you can access with an account:

Home function brings a user back to the initial page, which introduces the scope and tools of the application and presents a tutorial on how to navigate the application.

Log out function results in going back to the guest Welcome page, with the user no longer logged in their account.

Settings function allows the user to make any changes to their personal profile.

  • General settings
  • Manage your children
  • Change password

Memories function displays in a pleasing way the journal-entries-like blocks, each containing a picture and a relevant description.

  • Choose a child from the leftbar menu
  • Add a memory
  • Edit a memory
  • Delete a memory
  • Share a memory on the Social Media platforms

Calendars function offers the user a way to organize their children's feeding and sleeping timetables.

  • Choose a child from the leftbar menu
  • Add a new entry in the calendar
  • Edit the time or a note
  • Mark an activity as checked/unchecked

Friends function presents to the user any relationship that their children might have.

  • Choose a child from the leftbar menu
  • Add a new friend
  • Edit a relationship
  • Delete a relationship

Medical function saves the medical history of the user's children.

  • Choose a child from the leftbar menu
  • Add a new record
  • Edit a record
  • Delete a record

2.2. User classes and Characteristics

The Web application "Baby Manager" is intended for the parents and parents-to-be who are looking for management and organizing features for their children's journeys.

A guest user is a person that has their first encounter with the application or a person that is not logged in. They have access to the "Welcome" page, where they can learn information about what they could benefit once they create an account. The page is designed so that the guest user will be well documented about the features and their uses, as to make a right decision whether or not to join. A guest user also has the possibility to sign up and become a logged user. By trying to log in without an account already created, the user will get a warning about their credentials not being in the database. A guest user doesn't have access to any of the actual features for managing their children's information. Parents or soon to be parents that are searching for applications about raising children may fit in this category.

A logged user is a person that has previously created an account by signing up and used their credentials to log in their said account.
